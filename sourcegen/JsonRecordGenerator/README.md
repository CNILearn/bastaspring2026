# JsonRecordGenerator

A C# source generator that creates record classes from JSON files at compile time. This generator reads JSON files and automatically generates corresponding C# record types with properties matching the JSON structure, including support for nested objects and arrays.

## Features

- 🔥 **Compile-time JSON to Record Generation**: Processes JSON files during build and generates C# records
- 🏗️ **Nested Object Support**: Automatically creates nested record types for complex JSON structures  
- 📋 **Array/Collection Support**: Handles arrays and generates `IReadOnlyList<T>` properties
- ⚙️ **Configurable**: Customize namespaces, property naming conventions, access modifiers, and record names
- 🎯 **Type-safe**: Generated records provide compile-time type safety for your JSON data
- 🚀 **Modern C#**: Uses .NET 9 features and modern C# record syntax

## Quick Start

### 1. Add JSON Files to Your Project

Create JSON files and add them as `AdditionalFiles` in your project:

```xml
<ItemGroup>
  <AdditionalFiles Include="data.json" />
  <AdditionalFiles Include="config.json" />
</ItemGroup>
```

### 2. Mark Classes for Generation

Use the `[JsonRecord]` attribute to mark classes that should be generated from JSON:

```csharp
using JsonRecordGenerator.Attributes;

[JsonRecord("data.json")]
public partial class UserData
{
}
```

### 3. Use the Generated Records

The generator creates records with properties matching your JSON structure:

```csharp
// For JSON: {"id": 123, "name": "John", "isActive": true}
var user = new UserData 
{
    Id = 123,
    Name = "John", 
    IsActive = true
};
```

## JSON Examples and Generated Code

### Basic JSON Structure

**Input JSON (`user.json`):**
```json
{
  "id": 123,
  "name": "John Doe",
  "email": "john@example.com",
  "age": 30,
  "isActive": true,
  "balance": 1250.75
}
```

**Generated C# Record:**
```csharp
// <auto-generated/>
#nullable enable
using System;
using System.Collections.Generic;

namespace MyApp;

public record UserData
{
    public int Id { get; init; }
    public string? Name { get; init; }
    public string? Email { get; init; }
    public int Age { get; init; }
    public bool IsActive { get; init; }
    public double Balance { get; init; }
}
```

### Nested Objects

**Input JSON (`profile.json`):**
```json
{
  "user": {
    "id": 456,
    "profile": {
      "firstName": "Jane",
      "lastName": "Smith",
      "preferences": {
        "theme": "dark",
        "notifications": true
      }
    }
  },
  "metadata": {
    "created": "2024-01-15T10:30:00Z",
    "version": "1.0.0"
  }
}
```

**Generated C# Records:**
```csharp
public record ProfileData
{
    public User User { get; init; }
    public Metadata Metadata { get; init; }
}

public record User
{
    public int Id { get; init; }
    public Profile Profile { get; init; }
}

public record Profile
{
    public string? FirstName { get; init; }
    public string? LastName { get; init; }
    public Preferences Preferences { get; init; }
}

public record Preferences
{
    public string? Theme { get; init; }
    public bool Notifications { get; init; }
}

public record Metadata
{
    public string? Created { get; init; }
    public string? Version { get; init; }
}
```

### Arrays and Collections

**Input JSON (`products.json`):**
```json
{
  "products": [
    {
      "id": 1,
      "name": "Laptop",
      "price": 999.99
    },
    {
      "id": 2,
      "name": "Mouse", 
      "price": 29.99
    }
  ],
  "categories": ["electronics", "computers"],
  "store": {
    "name": "Tech Store",
    "location": "Downtown"
  }
}
```

**Generated C# Records:**
```csharp
public record ProductData
{
    public IReadOnlyList<Products> Products { get; init; }
    public IReadOnlyList<string?> Categories { get; init; }
    public Store Store { get; init; }
}

public record Products
{
    public int Id { get; init; }
    public string? Name { get; init; }
    public double Price { get; init; }
}

public record Store
{
    public string? Name { get; init; }
    public string? Location { get; init; }
}
```

## Configuration Options

### JsonRecordAttribute Properties

- **Namespace**: Custom namespace for generated records
- **RecordName**: Custom name for the main record (overrides class name)
- **PropertyNamingConvention**: Controls property naming
  - `PascalCase` (default): `firstName` → `FirstName`
  - `CamelCase`: `firstName` → `firstName`  
  - `KeepOriginal`: `firstName` → `firstName`
- **AccessModifier**: Controls record accessibility (`public`, `internal`, etc.)

### Examples with Configuration

```csharp
// Custom namespace and record name
[JsonRecord("data.json", 
    Namespace = "MyApp.Models",
    RecordName = "PersonInfo")]
public partial class UserData { }

// Custom naming convention
[JsonRecord("api-response.json",
    PropertyNamingConvention = PropertyNamingConvention.CamelCase)]
public partial class ApiResponse { }

// Internal access modifier
[JsonRecord("internal-config.json",
    AccessModifier = "internal",
    Namespace = "MyApp.Internal")]
public partial class InternalConfig { }

// All options combined
[JsonRecord("complex-data.json",
    Namespace = "MyApp.Generated.Models",
    RecordName = "ComplexDataStructure", 
    PropertyNamingConvention = PropertyNamingConvention.PascalCase,
    AccessModifier = "public")]
public partial class ComplexData { }
```

## Supported JSON Types

| JSON Type | C# Type | Notes |
|-----------|---------|-------|
| `string` | `string?` | Nullable by default |
| `number` (integer) | `int` | Detects integers vs decimals |
| `number` (decimal) | `double` | For floating-point numbers |
| `boolean` | `bool` | Direct mapping |
| `object` | Custom record | Creates nested record type |
| `array` | `IReadOnlyList<T>` | Type T determined from array elements |
| `null` | `object?` | Fallback for null values |

## Build Integration

Add the generator as an analyzer to your project:

```xml
<ItemGroup>
  <ProjectReference Include="path/to/JsonRecordGenerator.csproj"
      OutputItemType="Analyzer" 
      ReferenceOutputAssembly="false" />
</ItemGroup>

<ItemGroup>
  <AdditionalFiles Include="your-data.json" />
  <AdditionalFiles Include="other-file.json" />
</ItemGroup>
```

## Requirements

- .NET 9.0 or later
- C# 12.0 or later
- MSBuild-based projects (.NET SDK style)

## How It Works

1. **Build Time**: The source generator analyzes your code for `[JsonRecord]` attributes
2. **JSON Processing**: Reads the specified JSON files from `AdditionalFiles`
3. **Code Generation**: Creates C# record types matching the JSON structure
4. **Type Safety**: Generated records provide compile-time checking and IntelliSense

## Benefits

- **🚀 Performance**: No runtime JSON parsing overhead
- **🛡️ Type Safety**: Compile-time checking prevents runtime errors
- **📝 IntelliSense**: Full IDE support with autocomplete and refactoring
- **🔄 Maintainability**: Records automatically update when JSON changes
- **⚡ Fast Development**: Rapid prototyping with JSON data sources

## Error Handling

The generator provides helpful diagnostics:

- **JRG001**: JSON Record Generation Error - General generation errors
- **JRG002**: JSON File Not Found - Missing JSON files
- **JRG003**: JSON Parse Error - Invalid JSON syntax

## Limitations

- JSON files must be valid at build time
- Complex JSON schemas may require manual type adjustments
- Generated types are immutable records (by design)
- Circular references in JSON are not supported
